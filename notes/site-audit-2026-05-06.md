# Аудит сайту (2026-05-06)

## Що перевірено
- Підозрілість/дорвей-ознаки (масові шаблонні сторінки, thin content, дублювання).
- «Зайвий» або низькоякісний текст.
- Наявність зображень Unsplash у постах.
- Коректність зображень (биті локальні шляхи, lazy-loading).
- Базова mobile-оптимізація (наявність viewport).

## Результати

### 1) Підозрілість та ШІ-дорвей сигнали
- Виявлено дуже велику кількість однотипних сторінок (`476` HTML-сторінок; значна частина — мультимовні варіанти профілів і стартапів).
- Авто-аудит контенту показав **thin content** на низці сторінок (404/about/cookies/privacy/terms та частина startup).
- Це не є прямим доказом дорвею, але є **ризиком SEO-класифікації як шаблонного/масово згенерованого контенту**, якщо не посилити унікальність, джерела й редакторську цінність.

### 2) Непотрібний/слабкий текст
- Найбільша проблема — короткі сторінки службового типу (404/legal/about) та окремі короткі startup-сторінки.
- Рекомендація: додати унікальні блоки (пояснення методології, джерела, оновлення, контекст локального ринку).

### 3) Unsplash у кожному пості
- Перевірено startup-пости: `200` файлів `startup-*.html`.
- Усі мають хоча б одне зображення.
- Лише `8` із `200` мають посилання на `images.unsplash.com`.
- Висновок: **ні, Unsplash є не в кожному пості**.

### 4) Коректність зображень
- Скрипт перевірки не знайшов битих локальних шляхів до зображень.
- На сторінках із `<img>` застосовано `loading="lazy"`.
- Частина зображень береться з зовнішніх CDN (Unsplash/UI Avatars), що може впливати на стабільність/швидкість; у кількох місцях є fallback через `onerror`.

### 5) Mobile-оптимізація
- У перевірених HTML-файлах відсутній мета-тег viewport (`name="viewport"`) — критичний базовий елемент адаптивності.
- Навіть за наявності responsive CSS, без viewport мобільний UX зазвичай погіршується.

## Пріоритетні дії
1. Додати `<meta name="viewport" content="width=device-width, initial-scale=1">` у базовий шаблон.
2. Посилити унікальність і глибину контенту на thin-сторінках.
3. Зменшити залежність від зовнішніх image-CDN для core-сторінок (кеш/локальні копії критичних зображень).
4. Додати редакторські сигнали якості: автор, дата оновлення, джерела, метод фактчеку.

## Команди, які запускались
- `node scripts/audit-content.mjs`
- `rg -n "unsplash|source.unsplash|images.unsplash" src/pages src/assets src/templates`
- Node one-liner для підрахунку startup-сторінок, зображень і viewport.
